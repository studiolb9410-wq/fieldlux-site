Features

Every metric a TD actually checks.

Not a viewer. A scoring engine that runs the same math a careful installer would — only faster, and consistent run-to-run.

⚡

Judgment engine

C-metrics scored 0–100 across coverage, brightness, pixel density, spill, overscan, blend uniformity, seam stability. PASS / AT_RISK / NOT_RECOMMENDED with root cause.

🔴

Stress analysis

S-scenarios perturb the rig — lateral drift, distance drift, yaw, pitch, reflectance. One resilience score summarises how much the install can absorb before scores crater.

💡

Prescription engine

Not just what's wrong — what to do. "Move Projector 2 back 0.3 m" with estimated score uplift. Fix in the app, re-analyze, ship.

02

Run analysis

One click. Photometrics, coverage, stress scenarios, blend zones, audience view — results in under three seconds.

03

Ship with confidence

Go / No-Go verdict, resilience score, prescribed fixes. Share a read-only 3D link with the client or export the PDF tech rider.

Design principles

Why it's built the way it is.

Three commitments that shape every decision, from the shader up to the PDF.

01 Physics first

Every score ties back to optical math — inverse-square intensity, Lambert falloff, pixel pitch, real lens profiles. The app won't give you a verdict it can't justify from first principles.

02 Show the work

Confidence bands, error budgets, simulation disclaimers are on the face of every report. You know exactly what's estimated vs measured, and how much margin sits around each number.

03 Hand-off shaped

A preflight tool is only useful if the output travels well. PDFs, share links, JSON handoffs, calibration profiles — shaped for how the rigging crew, the client, and the downstream pipeline actually work.
